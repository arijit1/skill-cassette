# Docs Style Guide

- Keep examples executable and copy-paste friendly.
- Prefer short headings and direct language.
- Use bullets when they make the structure easier to scan.
- Match the repo voice instead of adding marketing fluff inside technical docs.
- When updating a README, keep the quickstart first and the caveats last.
- If a command or snippet changes, make the example line up with the CLI contract.
- Favor a small amount of context over a large wall of prose.
